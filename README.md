**Введение**

В практике часто возникает задача поиска минимума функции нескольких переменных. Классические методы оптимизации (градиентный спуск, метод Ньютона) требуют вычисления производных, что не всегда возможно для сложных функций.

В данной работе реализован метод Нелдера-Мида - прямой метод оптимизации, не использующий производные. Программа написана на языке C# с соблюдением принципов программирования

**Цель работы**

1.  Реализовать метод Нелдера-Мида для безусловной оптимизации функций.
2.  Сделать реализацию универсальной - для произвольной размерности пространства (от 2 до N измерений).
3.  Протестировать метод на различных тестовых функциях.
4.  Оформить код с соблюдением принципов чистого кода.

**Теоретический материал**

**Метод Нелдера-Мида**

Метод Нелдера-Мида (симплекс-метод) — это метод безусловной оптимизации, основанный на геометрических преобразованиях симплекса.

**Симплекс** - геометрическая фигура в N-мерном пространстве, состоящая из N+1 точки. Для двумерного случая - треугольник, для трехмерного - тетраэдр.

**Основная идея**: на каждой итерации алгоритм замещает наихудшую точку симплекса новой точкой, полученной с помощью операций:

- отражение
- растяжение
- сжатие
- редукция

### Параметры метода

| **Параметр** | **Стандартное значение** | **Назначение** |
| --- | --- | --- |
| α (альфа) | 1.0 | Коэффициент отражения |
| β (бета) | 2.0 | Коэффициент растяжения |
| γ (гамма) | 0.5 | Коэффициент сжатия |
| σ (сигма) | 0.5 | Коэффициент редукции |

**Описание алгоритма Нелдера-Мида**

Метод Нелдера-Мида работает следующим образом:

**Шаг 1.** Создается начальный симплекс - для N измерений нужно N+1 точек. Например, для двумерного случая это треугольник.

**Шаг 2.** В каждой точке вычисляется значение целевой функции. Точки сортируются от лучшей (наименьшее значение) до худшей (наибольшее значение).

**Шаг 3.** На каждой итерации:

- Находится центр тяжести всех точек, кроме худшей.
- Худшая точка отражается относительно этого центра - получается новая точка.

**Шаг 4.** Дальнейшие действия зависят от того, насколько хороша отраженная точка:

- Если она **лучше лучшей** - делают растяжение (идут еще дальше).
- Если **между лучшей и средней** - просто принимают отражение.
- Если **хуже средней** - пробуют сжатие (внутреннее или внешнее).
- Если сжатие не помогло - сжимают весь симплекс к лучшей точке (редукция).

**Шаг 5.** Заменяют худшую точку на лучшую из полученных. Снова сортируют точки.

**Шаг 6.** Проверяют условие остановки: если значения функции во всех точках стали достаточно близки - алгоритм завершается. Иначе - переход на шаг 3.

**Условие остановки**

Алгоритм завершается при выполнении одного из условий:

1.  Разность между максимальным и минимальным значениями функции в точках симплекса меньше заданной точности (в коде - tolerance)
2.  Достигнуто максимальное количество итераций (в коде - maxIterations)

**Тестовые функции и их свойства**

Для тестирования метода использовались следующие функции:

**1\. Сферическая функция**

**Формула**: f(x₁, x₂, ..., x<sub>n</sub>) = Σxᵢ²

Свойства:

- Гладкая, выпуклая функция
- Единственный глобальный минимум: в точке (0, 0, ..., 0)
- Значение в минимуме: 0

**2\. Функция Розенброка**

**Формула**: f(x, y) = (1 - x)² + 100·(y - x²)²

Свойства:

- Невыпуклая функция, имеет узкий, сильно изогнутый овраг
- Глобальный минимум: в точке (1, 1)
- Значение в минимуме: 0

**3\. Функция Растригина**

**Формула:** f(x, y) = 20 + x² - 10·cos(2πx) + y² - 10·cos(2πy)

Свойства:

- Невыпуклая функция, имеет множество локальных минимумов
- Глобальный минимум: в точке (0, 0)
- Значение в минимуме: 0

**4\. Функция Аккли**

**Формула:** f(x, y) = -20·exp(-0.2·√(0.5·(x²+y²))) - exp(0.5·(cos(2πx)+cos(2πy))) + e + 20

Свойства:

- Невыпуклая функция, много локальных минимумов, экспоненциальная
- Глобальный минимум: в точке (0, 0)
- Значение в минимуме: 0

**5\. Функция Била**

**Формула:** f(x, y) = (1.5 - x + xy)² + (2.25 - x + xy²)² + (2.625 - x + xy³)²

Свойства:

- Невыпуклая функция, имеет длинный плоский овраг
- Глобальный минимум: в точке (3, 0.5)
- Значение в минимуме: 0

**6\. Функция Бута**

**Формула:** f(x, y) = (x + 2y - 7)² + (2x + y - 5)²

Свойства:

- Квадратичная, выпуклая функция
- Глобальный минимум: в точке (1, 3)
- Значение в минимуме: 0

**7\. Функция Гольдштейна-Прайса**

**Формула:** f(x, y) = \[1 + (x + y + 1)² · (19 - 14x + 3x² - 14y + 6xy + 3y²)\] · \[30 + (2x - 3y)² · (18 - 32x + 12x² + 48y - 36xy + 27y²)\]

Свойства:

- Невыпуклая функция, несколько локальных минимумов
- Глобальный минимум: в точке (0, -1)
- Значение в минимуме: 3

**8\. Функция Химмельблау**

**Формула:** f(x, y) = (x² + y - 11)² + (x + y² - 7)²

Свойства:

- Невыпуклая функция, имеет 4 одинаковых глобальных минимума
- Глобальные минимумы: (3,2), (-2.805, 3.131), (-3.779, -3.283), (3.584, -1.848)
- Значение в минимуме: 0

**9\. Квадратичная функция**

**Формула:** f(x, y) = x² + xy + y² - 6x - 9y

Свойства:

- Гладкая, квадратичная
- Глобальный минимум: в точке (1, 4)
- Значение в минимуме: -21

**Вывод:**  
Метод Нелдера-Мида — это простой и эффективный способ оптимизации, не требующий вычисления производных. Благодаря этому он широко применяется на практике. Однако его недостаток - чувствительность к локальным минимумам: если у функции много локальных экстремумов метод может не найти глобальный минимум.

В ходе работы также было проведено исследование влияния параметров метода на сходимость и качество решения. Экспериментально подтверждено, что классические значения параметров (α = 1.0, β = 2.0, γ = 0.5, σ = 0.5) являются оптимальными для широкого круга задач. Отклонение от этих значений в ту или иную сторону приводит к увеличению количества итераций или ухудшению качества найденного решения. Однако на мультимодальных функциях (например: Растригина, Аккли) может наблюдаться отклонение: уменьшение коэффициента редукции σ с 0.5 до 0.1 сокращает количество итераций без ухудшения найденного решения. Это означает, что стандартный набор параметров хорошо подходит для всех функций, но для некоторых адаптация параметров может улучшить результат.
